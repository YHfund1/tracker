# 中东跟踪工程技能（从零版）

## 适用场景

当用户满足任一条件时使用本技能：

- 还没有项目，需要从零创建“中东地缘跟踪”网站工程。
- 已有项目，但结构不完整，需要补齐标准页面与数据管线。
- 需要执行每日/每次全量更新（数据+页面+校验）。

## 安全边界

- 默认只做本地更新。
- 不读取、不写入、不复用任何账号密码、Token、SSH、公钥、远程仓库地址。
- 不自动执行 `git commit`、`git push`、部署命令，除非用户单独明确要求。
- 发现工作区有无关脏改动时，不回滚用户文件。

## 执行总流程

1. 先读 [references/01-从零初始化.md](references/01-从零初始化.md) 判断是否需要建项目骨架。
2. 按 [references/02-全量更新流程.md](references/02-全量更新流程.md) 执行脚本更新与人工编辑更新。
3. 按 [references/03-模块编辑规范.md](references/03-模块编辑规范.md) 处理简报、战局、央行、供应链、研究、原油图谱。
4. 按 [references/04-校验与交付.md](references/04-校验与交付.md) 完成本地验收并输出结果。

## 关键原则

- 能更新 JSON 就不要重写页面外壳。
- 能复用现有脚本就不要新建永久脚本。
- 必须使用绝对日期，不用含糊“今天/昨天”。
- 对无法确认的事实要标注限制，不要编造。
